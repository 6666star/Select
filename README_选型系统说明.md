## 1. 系统概述

本系统是一套完整的**矿井主通风机变频选型计算软件**，包含命令行计算后端和 Web 可视化界面两部分。适用于具有容易时期和困难时期两种工况的矿井通风系统。

### 整体架构

```
┌────────────────────────────────────────────────────────────┐
│                    app.py（Web UI）                         │
│   参数输入 → 计算结果 → 选型对比 → 特性曲线 → 计算过程      │
└──────┬──────────┬──────────┬──────────┬────────────────────┘
       │          │          │          │
       ▼          ▼          ▼          ▼
  ┌─────────┐ ┌────────┐ ┌────────┐ ┌────────┐
  │  calc_  │ │  calc_ │ │selector│ │plot_res│
  │ engine  │ │engine_ │ │  .py   │ │  .py   │
  │  .py    │ │range.py│ │        │ │        │
  └─────────┘ └────────┘ └───┬────┘ └────────┘
                              │
                         ┌────▼────┐
                         │ fan_db  │
                         │  .py    │
                         └─────────┘
```

### 核心功能


| 功能           | 说明                                                       |
| -------------- | ---------------------------------------------------------- |
| 双工况通风计算 | 分别计算容易/困难时期的 Q、H、R、N                         |
| 变频风机选型   | 基于相似律，从实测数据库中自动匹配最优机型                 |
| 五步选型算法   | 范围初筛 → 工作点求解 → 效率验证 → 变频校核 → 综合评分 |
| 特性曲线绘图   | H-Q / η-Q / N-Q 三组曲线，含额定+变频+管道阻力线          |
| Web 可视化界面 | Streamlit 构建，浏览器操作，无需编写代码                   |

## 4. 模块详细说明

### 4.1 calc_engine.py — 单工况计算引擎

**职责**：从矿井基础参数计算单个工况下的通风机工作风量、工作风压、阻力系数和估算功率。

**输入**：


| 参数         | 类型  | 单位    | 说明                       |
| ------------ | ----- | ------- | -------------------------- |
| `Q1`         | float | m³/min | 各采掘工作面总需风量       |
| `h_f`        | float | Pa      | 通风网路总阻力             |
| `Q2`         | float | m³/min | 其他用风地点风量（默认 0） |
| `h_vc`       | float | Pa      | 出口速度压力损失（默认 0） |
| `h_e`        | float | Pa      | 附加损失（默认 0）         |
| `altitude_m` | float | m       | 矿井海拔高度（默认 0）     |

**选型系数**（`SelectionCoeffs`）：


| 系数    | 默认值 | 公式用途                        |
| ------- | ------ | ------------------------------- |
| `K`     | 1.20   | `Q_总 = K × (Q1 + Q2)`         |
| `K_Q`   | 1.15   | `Q_f = K_Q × Q_总`             |
| `K_H`   | 1.10   | `H_f = K_H × H_总 × ρ_ratio` |
| `K_N`   | 1.20   | `N_motor = K_N × N_shaft`      |
| `eta_f` | 0.75   | 估算用全压效率                  |
| `eta_t` | 0.95   | 传动效率                        |

**计算流程**：

```
海拔修正 → 总需风量 → 工作风量 → 总压力 → 工作风压 → 阻力系数 → 轴功率 → 电机功率
```


| 步骤     | 公式                                          |
| -------- | --------------------------------------------- |
| 海拔修正 | `ρ/ρ₀ = exp(-h/8500)`                      |
| 总需风量 | `Q_总 = K × (Q1 + Q2)`                       |
| 工作风量 | `Q_f = K_Q × Q_总`，同时输出 m³/s           |
| 工作风压 | `H_f = K_H × (h_f + h_vc + h_e) × ρ_ratio` |
| 阻力系数 | `R = H_f / Q_f²`                             |
| 轴功率   | `N = Q_f × H_f / (1000 × η_f × η_t)`     |
| 电机功率 | `N_motor = K_N × N`                          |

**输出**（`EngineResult`）：

```python
Q_f_ms: float    # 工作风量 (m³/s)
H_f: float       # 工作风压 (Pa)
R: float         # 阻力系数 (N·s²/m⁸)
N_shaft: float   # 估算轴功率 (kW)
N_motor: float   # 配套电机功率 (kW)
```

### 4.2 calc_engine_range.py — 双工况计算引擎

**职责**：分别计算矿井容易时期与困难时期两个工况，是选型模块的前置数据源。

**输入**（`DualPeriodParams`）：

```python
Q1_easy: float     # 容易时期需风量 (m³/min)
h_f_easy: float    # 容易时期网路总阻力 (Pa)
Q1_hard: float     # 困难时期需风量 (m³/min)
h_f_hard: float    # 困难时期网路总阻力 (Pa)
```

**处理逻辑**：

1. 对容易时期调用 `calc_engine.run()` → `EngineResult`
2. 对困难时期调用 `calc_engine.run()` → `EngineResult`
3. 由各时期的 R 和 Q_f 生成管道特性曲线数据 `H = R × Q²`

**输出**（`DualPeriodResult`）：

```python
easy: EngineResult       # 容易时期结果
hard: EngineResult       # 困难时期结果
easy_curve: tuple        # (Q_list, H_list) 管道曲线
hard_curve: tuple        # (Q_list, H_list) 管道曲线
```

选型模块使用的关键属性：`dual.easy.Q_f_ms`、`dual.easy.H_f`、`dual.easy.R`，以及困难时期的对应值。

### 4.3 fan_db.py — 风机数据库

**职责**：定义风机数据结构，管理内置风机库（5 款实测数据机型），构建多项式拟合插值对象。

**设计原则**：

- 只接受实测数据，不支持两端点近似生成（精度不足，工程不可靠）
- 唯一创建入口：`make_fan_from_data()`
- `FAN_DB` 列表供选型模块自动读取

#### 数据结构

```python
@dataclass
class FanCurvePoint:
    Q: float     # 风量 (m³/s)
    H: float     # 全压 (Pa)
    eta: float   # 全压效率 (0~1)

@dataclass
class FanModel:
    model_id: str          # 型号
    series: str            # 系列
    fan_type: str          # "counter_rotating" 对旋式
    rated_rpm: float       # 额定转速 (r/min)
    rpm_range: tuple       # (n_min, n_max) 允许调速范围
    motor_kw: float        # 单台电机功率 (kW)
    motor_count: int       # 电机台数
    curve_points: list     # List[FanCurvePoint]
```

#### 内置数据库（FAN_DB）


| 序号 | 型号              | 额定转速  | 装机功率  | 数据点 | 数据来源     |
| ---- | ----------------- | --------- | --------- | ------ | ------------ |
| 1    | FBCDZ-6No23/2x315 | 990 r/min | 2×315 kW | 39     | 出厂检测报告 |
| 2    | FBCDZ(B)-6-No20   | 980 r/min | 2×220 kW | 10     | 厂家性能数据 |
| 3    | FBCDZ(B)-6-No22   | 980 r/min | 2×315 kW | 10     | 厂家性能数据 |
| 4    | FBCDZ(B)-8-No24   | 740 r/min | 2×185 kW | 10     | 厂家性能数据 |
| 5    | FBCDZ(B)-8-No28   | 740 r/min | 2×355 kW | 10     | 厂家性能数据 |

#### 曲线拟合方法

`build_fan_interpolated()` 使用 `numpy.polyfit` 最小二乘多项式拟合：


| 曲线 | 拟合阶次 | 模型                                 | 说明                  |
| ---- | -------- | ------------------------------------ | --------------------- |
| H-Q  | 2 次     | `H = a₀ + a₁·Q + a₂·Q²`        | 标准风机二次 H-Q 曲线 |
| η-Q | 4 次     | `η = b₀ + b₁·Q + ... + b₄·Q⁴` | 捕捉钟形效率曲线      |

选择 polyfit 而非 CubicSpline 的原因：实测数据含微小噪声，CubicSpline 强制过每个点会产生不自然波动，polyfit 最小二乘拟合更鲁棒光滑。

### 4.4 selector.py — 风机精细选型核心模块

**职责**：对风机数据库中的候选型号依次执行五步选型算法，输出按综合评分排序的推荐列表。这是整个系统的核心模块。

**入口函数**：

```python
def run_selector(dual_result, eta_min=0.70, eta_t=0.95, fans=None) -> SelectorResult
```

**输出数据结构**：

```python
@dataclass
class SelectionCandidate:
    fan: FanModel            # 风机型号
    Q_op_easy: float         # 容易时期工作风量 (m³/s)
    H_op_easy: float         # 容易时期工作风压 (Pa)
    Q_op_hard: float         # 困难时期工作风量 (m³/s)
    H_op_hard: float         # 困难时期工作风压 (Pa)
    eta_easy: float          # 容易时期效率
    eta_hard: float          # 困难时期效率
    N_shaft_easy: float      # 容易时期轴功率 (kW)
    N_shaft_hard: float      # 困难时期轴功率 (kW)
    N_motor_req: float       # 建议最小电机功率 (kW)
    n_ratio: float           # 转速比 n_hard/n_easy
    n_easy_rpm: float        # 容易时期转速 (r/min)
    n_hard_rpm: float        # 困难时期转速 (r/min)
    vfd_error: float         # 相似律误差 ε (%)

@dataclass

    candidates: list         # List[SelectionCandidate] 按 score 降序
    best: SelectionCandidate # 最优推荐
    screened_out: int        # 初筛淘汰数
    rejected_eff: int        # 效率/功率淘汰数
    steps: list              # 完整计算过程记录
```

## 5. 核心算法详解

### 5.1 管道特性曲线与阻力系数

矿井通风管道的阻力特性遵循平方定律：

```
H = R × Q²
```

- `H` — 风压 (Pa)
- `Q` — 风量 (m³/s)
- `R` — 管道阻力系数 (N·s²/m⁸)

阻力系数由工况反算：`R = H_f / Q_f²`。

容易时期和困难时期的 R 值通常不同（矿井开采阶段不同，巷道结构变化），这意味着两条管道阻力抛物线形状不同。

---

### 5.2 风机 H-Q 曲线拟合方法

**H-Q 曲线**（二次多项式，标准风机物理模型）：

```
H(Q) = a₀ + a₁·Q + a₂·Q²
```

**η-Q 曲线**（四次多项式，捕捉钟形效率曲线）：

```
η(Q) = b₀ + b₁·Q + b₂·Q² + b₃·Q³ + b₄·Q⁴
```

拟合过程：

```python
import numpy as np
H_coeffs   = np.polyfit(Q_data, H_data,   2)   # 二次拟合
eta_coeffs = np.polyfit(Q_data, eta_data, 4)   # 四次拟合
H_poly   = np.poly1d(H_coeffs)                 # 可调用对象
eta_poly = np.poly1d(eta_coeffs)
```

`poly1d` 对象支持标量和数组输入，η 值自动截断到 [0, 1]。

### 5.3 变频工作点求解（VFD 相似律）

这是本系统最核心的算法。

#### 风机相似律

转速从 n₀ 变为 n 时：

```
Q' = Q × (n/n₀)          风量与转速成正比
H' = H × (n/n₀)²         风压与转速平方成正比
η' = η                    效率不变
N' = N × (n/n₀)³         功率与转速三次方成正比
```

#### 求解步骤

**问题**：已知设计目标 (Q_req, H_req) 和管道阻力 R，求变频转速 n_op。

**Step 1 — 求额定曲线上的基准点 u**

```
解方程：H_fan(u) = R · u²
方法：brentq 二分法求零点
```

物理含义：如果保持额定转速不变，在阻力 R 的管道中，风机自然平衡在流量 u 处。

**Step 2 — 计算变频转速**

```
n_op = n₀ × Q_req / u
```

物理含义：将基准点 (u, H_fan(u)) 按相似律缩放到目标点 (Q_req, H_req)。

**Step 3 — 验证风压自动满足**

```
H' = H_fan(u) × (Q_req/u)² = R·u² × (Q_req/u)² = R·Q_req² = H_req  ✓
```

风量和风压同时精确满足，这是恒等式，不是近似。

**Step 4 — 读取效率**

```
η_op = η_fan(u)    （相似律保证基准点与目标点效率相同）
```

**Step 5 — 计算功率**

```
N_shaft = Q_req × H_req / (1000 × η_op × η_t)
```

功率计算使用实际工作点的 Q 和 H。

#### 图解

```
    H (Pa)
    │
    │   ╲  额定 H-Q 曲线
    │    ╲
    │ 基准点 ●──── H_fan(u) = R·u²    ← brentq 求得
    │       ╱╲
    │      ╱  ╲   变频后 H-Q 曲线
    │     ╱    ╲
    │ 目标点 ●──── R·Q_req² = H_req    ← 自动满足
    │   ╱       ╲
    │  ╱  H=R·Q²  ╲
    └───────────────── Q (m³/s)
        Q_req    u
```

### 5.4 五步选型算法

```
步骤①  screen_by_range()     快速范围初筛
  │     筛除 Q/H/功率明显不匹配的型号
  ▼
步骤②  solve_working_point()  管道-风机交点求解
  │     brentq 求根：H_fan(u) = R·u²
  │     计算变频转速 n = n₀ × Q_req / u
  │     检查转速是否在 rpm_range 范围内
  ▼
步骤③  verify_efficiency()    效率与功率验证
  │     η 从基准点读取（相似律保证）
  │     N_shaft = Q × H / (1000 × η × η_t)
  │     检查 η ≥ η_min，装机功率 ≥ N_motor_req
  ▼
步骤④  verify_vfd_speed()     变频转速比验证
  │     n_ratio = n_hard / n_easy
  │     ε = |H_h/H_e - n_ratio²| / n_ratio² × 100%
  ▼
步骤⑤  score_candidate()      综合评分排序
```

#### 步骤① 范围初筛


| 条件              | 说明                           |
| ----------------- | ------------------------------ |
| `Q_max ≥ Q_hard` | 风机最大额定风量覆盖困难时期   |
| `H_max ≥ H_hard` | 风机最大额定风压覆盖困难时期   |
| `Q_min ≤ Q_hard` | 工作点在风机覆盖范围内         |
| 功率范围          | 装机功率在估算值的 50%~300% 内 |

#### 步骤② 工作点求解

对通过初筛的每台风机，分别求解容易/困难时期的基准点 u 和变频转速 n。若无交点或转速超出 `rpm_range`，该型号淘汰。

#### 步骤③ 效率验证

- `η_easy ≥ η_min` 且 `η_hard ≥ η_min`
- `fan.total_motor_kw ≥ max(N_easy, N_hard) × K_N`
- `K_N = 1.15`（GB 50215，大型矿用主通风机）

#### 步骤④ 变频校核

相似律误差评级：


| ε 范围 | 评级 | 含义           |
| ------- | ---- | -------------- |
| < 3%    | 优秀 | 两工况完美匹配 |
| 3%~8%   | 良好 | 变频效果好     |
| > 8%    | 较差 | 建议复核       |

#### 步骤⑤ 综合评分

详见[第 5.5 节](#55-综合评分公式)。

---

### 5.5 综合评分公式

```
score = 0.4 × η̃_hard + 0.3 × η̃_easy + 0.2 × VFD̃ + 0.1 × motor̃
```


| 分项      | 公式                       | 权重        | 含义                   |
| --------- | -------------------------- | ----------- | ---------------------- |
| η̃_hard | `min(η_hard / 0.85, 1.0)` | 0.4         | 困难时期效率（最重要） |
| η̃_easy | `min(η_easy / 0.85, 1.0)` | 0.3         | 容易时期效率           |
| VFD̃     | `max(0, 1 - ε / 8.0)`     | 0.2         | 变频匹配度             |
| motor̃   | `max(0, 1 -                | 利用率-0.80 | /0.80)`                |

设计思路：困难时期权重最高（功耗最大），其次容易时期（运行时间最长），变频匹配度和电机利用率作为辅助指标。

## 10. 常见问题

### Q1：新加的风机为什么被淘汰了？

在 Web UI 的"选型对比"Tab 底部展开"被淘汰的机型"，可以看到每个被淘汰型号的具体原因。常见原因：

- 初筛阶段：Q_max 不够 / Q_min 过大 / H_max 不够
- 工作点阶段：额定曲线与管道阻力线无交点
- 转速阶段：所需变频转速超出 `rpm_range` 允许范围
- 效率阶段：工作点效率 < η_min
- 功率阶段：装机功率不足

### Q2：K 系数全部设为 1.0 是什么含义？

当输入的风量和风压已经是修正后的最终设计值时，K=1.0、K_Q=1.0、K_H=1.0 表示不再额外叠加系数。若输入原始需风量，应使用默认系数。

### Q3：变频转速范围为什么是 50%~100%？

矿用对旋风机配套变频器通常允许调速到额定转速的 50%。低于此值时气动性能急剧恶化、电机散热条件变差。

### Q4：如何修改设计工况参数？

- Web UI：直接在侧边栏修改数字即可
- 命令行：修改 `selector.py` 或 `plot_res.py` 的 `__main__` 块中的 `DualPeriodParams`

### Q5：为什么 H-Q 用 2 次拟合而不是更高次？

风机 H-Q 特性曲线的物理模型本身是二次抛物线（`H = H₀ - k·Q²`）。更高次会引入非物理拐点。η-Q 则需要 4 次来描述先升后降的钟形特征。

### Q6：相似律误差 ε 是什么含义？

ε 度量两个工况点是否接近同一条相似抛物线。ε=0 意味着两工况可以通过单纯调速完美切换；ε 越大说明两工况的管道特性差异越大，但只要 ε<8% 变频方案仍然可行。
